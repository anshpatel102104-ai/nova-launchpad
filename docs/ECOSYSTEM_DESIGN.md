# Launchpad в‡„ Bylda вЂ” Ecosystem Design System

**The core truth: Launchpad creates the business. Bylda runs the business.**

One intelligence layer, two distinct products. They share account, memory,
business context, decisions, and AI intelligence вЂ” they never share
navigation, home screens, shell, or mental model.

|         | Launchpad                        | Bylda                                |
| ------- | -------------------------------- | ------------------------------------ |
| Feeling | "I am building my business"      | "I am running my business"           |
| Model   | Linear, staged, mission-based    | Modular, operational, command center |
| Home    | Casefile mission control         | Operating system overview            |
| Accent  | Violet (`--primary` default)     | Cyan (`--bylda-accent`)              |
| Shell   | `LaunchpadSidebar` (staged rail) | `ByldaSidebar` (command rail)        |

## Non-negotiables (enforced in code)

1. **Never one shared dashboard.** `AppSidebar` is only a chooser; each
   product renders its own nav (`src/components/app/AppSidebar.tsx`).
2. **Never a generic post-onboarding home.** `/app/` resolves the landing
   by workspace mode (`src/routes/app.index.tsx` в†’ `resolveLandingPath`).
3. **Progress is visible.** Launchpad's spine is the five-stage bar; stages
   are _derived from real signals_ (tool runs, leads, won deals), never
   self-reported (`deriveLaunchpadProgress` in `src/lib/ecosystem.ts`).
4. **The closed loop is visible.** Cause-and-effect chips
   (`ClosedLoopChip`): _Saved to memory В· Generated by Bylda В· Next unlocked В·
   Updated from your last action В· Based on what Bylda learned_.
5. **Crossing products is a handoff, not a tab.** Launchpad shows the Bylda
   door at the bottom of its rail; when the build is proven it becomes the
   handoff hero: **"You built the business. Now run it."**

### Launchpad IA

- **Home** (`/app/mission-control`) вЂ” the casefile mission control:
  1. Stage bar (you are here) в†’ 2. Do this now (guided hero) в†’
  2. After this step (unlocks + one thing to fix) в†’
  3. Casefile (Bylda's take В· proven В· needs proof В· memory strip) в†’
  4. Your numbers.
- **Stages** вЂ” each stage links to where the work happens:
  Idea в†’ idea-validator В· Validate в†’ kill-my-idea В· Offer в†’ offer В·
  Build в†’ gtm-strategy В· Launch в†’ outcomes/launch В· Revenue в†’ first-customers.
- **Casefiles** (support, collapsed): Missions, Research, Assets, Memory.

### Bylda IA

- **Home** (`/app/bylda-home`) вЂ” the command center:
  health hero в†’ KPI blocks (Pipeline, Open, Won, Automations) в†’
  urgent attention tray в†’ pipeline snapshot в†’ next moves (AI, closed-loop) в†’
  right rail: automation status, what Bylda remembers, what changed.
- **Primary**: Home В· CRM В· Pipeline В· Automations В· Clients В· Tasks В· Reporting.
- **Operations** (support, collapsed): Inbox, Workflows, Campaigns,
  Knowledge, Activity, Integrations.
- **No coaching language.** Bylda never says "mission", "prove it", or
  stage-of-journey copy вЂ” it says _4 leads need attention_, _pipeline is live_.

## Stage derivation (proof, not opinion)

| Stage    | Done when                                                  |
| -------- | ---------------------------------------------------------- |
| Idea     | any successful tool run, or idea scored                    |
| Validate | idea validated + stress-tested (kill-my-idea / competitor) |
| Offer    | offer generated with pricing                               |
| Build    | customer plan / go-to-market system written                |
| Launch   | first leads saved                                          |
| Revenue  | first deal won                                             |

`readyForBylda` = Build done **and** Launch done в†’ the handoff hero appears and
the sidebar door upgrades from a quiet link to "Ready to operate".

## Product identity tokens

`[data-product="bylda"]` on the app shell (set in `src/routes/app.tsx` from
workspace mode) remaps the semantic accent tokens вЂ” `--primary`,
`--primary-soft/-border/-hover/-active/-glow`, `--ring`, `--accent`, sidebar
accents вЂ” to `--bylda-accent` (cyan, per-theme values in `styles.css`).
Components keep using `var(--primary)` and inherit the correct product
identity for free. Launchpad keeps the default violet.

## Mobile

Two tab bars (`MobileTabBar`), one per product, mirroring each sidebar:

- Launchpad: Home В· Missions В· Research В· Memory В· Ask
- Bylda: Home В· CRM В· Pipeline В· Tasks В· Ask

## Copy system

Concise, operational, plain. Say _"fix your offer"_, _"4 leads need
attention"_, _"pipeline is live"_ вЂ” never _"explore features"_ or
_"configure modules"_. Launchpad speaks in proof and momentum; Bylda speaks
in status and control.

## Validation test (run before shipping any screen)

Clarity in 10s В· unmistakably Launchpad **or** Bylda В· obvious next action В·
creates momentum В· closed loop visible В· adaptive not trendy В· beautiful on
a phone В· an ambitious 18-year-old instantly gets it В· doesn't look like
standard SaaS.
