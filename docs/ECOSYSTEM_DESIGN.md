# Launchpad ⇄ Nova — Ecosystem Design System

**The core truth: Launchpad creates the business. Nova runs the business.**

One intelligence layer, two distinct products. They share account, memory,
business context, decisions, and AI intelligence — they never share
navigation, home screens, shell, or mental model.

|         | Launchpad                        | Nova                                 |
| ------- | -------------------------------- | ------------------------------------ |
| Feeling | "I am building my business"      | "I am running my business"           |
| Model   | Linear, staged, mission-based    | Modular, operational, command center |
| Home    | Casefile mission control         | Operating system overview            |
| Accent  | Violet (`--primary` default)     | Cyan (`--nova-accent`)               |
| Shell   | `LaunchpadSidebar` (staged rail) | `NovaSidebar` (command rail)         |

## Non-negotiables (enforced in code)

1. **Never one shared dashboard.** `AppSidebar` is only a chooser; each
   product renders its own nav (`src/components/app/AppSidebar.tsx`).
2. **Never a generic post-onboarding home.** `/app/` resolves the landing
   by workspace mode (`src/routes/app.index.tsx` → `resolveLandingPath`).
3. **Progress is visible.** Launchpad's spine is the five-stage bar; stages
   are _derived from real signals_ (tool runs, leads, won deals), never
   self-reported (`deriveLaunchpadProgress` in `src/lib/ecosystem.ts`).
4. **The closed loop is visible.** Cause-and-effect chips
   (`ClosedLoopChip`): _Saved to memory · Generated by Nova · Next unlocked ·
   Updated from your last action · Based on what Nova learned_.
5. **Crossing products is a handoff, not a tab.** Launchpad shows the Nova
   door at the bottom of its rail; when the build is proven it becomes the
   handoff hero: **"You built the business. Now run it."**

## Architecture

```
src/lib/ecosystem.ts                 ← single source of truth for the split
├─ LAUNCHPAD_STAGES                  Idea → Validate → Offer → Build → Launch → Revenue
├─ deriveLaunchpadProgress(graph)    stage states, proof, needs-proof, readiness
├─ NOVA_PRIMARY_NAV / NOVA_SUPPORT_NAV
├─ LAUNCHPAD_SUPPORT_NAV             Missions · Research · Assets · Memory
└─ resolveLandingPath(mode)          post-onboarding routing

src/hooks/use-business-graph.ts      ← shared intelligence layer (unchanged)
src/hooks/use-workspace-mode.ts      ← create ↔ operate (workspaces.mode)
```

### Launchpad IA

- **Home** (`/app/mission-control`) — the casefile mission control:
  1. Stage bar (you are here) → 2. Do this now (guided hero) →
  2. After this step (unlocks + one thing to fix) →
  3. Casefile (Nova's take · proven · needs proof · memory strip) →
  4. Your numbers.
- **Stages** — each stage links to where the work happens:
  Idea → idea-validator · Validate → kill-my-idea · Offer → offer ·
  Build → gtm-strategy · Launch → outcomes/launch · Revenue → first-customers.
- **Casefiles** (support, collapsed): Missions, Research, Assets, Memory.

### Nova IA

- **Home** (`/app/nova-home`) — the command center:
  health hero → KPI blocks (Pipeline, Open, Won, Automations) →
  urgent attention tray → pipeline snapshot → next moves (AI, closed-loop) →
  right rail: automation status, what Nova remembers, what changed.
- **Primary**: Home · CRM · Pipeline · Automations · Clients · Tasks · Reporting.
- **Operations** (support, collapsed): Inbox, Workflows, Campaigns,
  Knowledge, Activity, Integrations.
- **No coaching language.** Nova never says "mission", "prove it", or
  stage-of-journey copy — it says _4 leads need attention_, _pipeline is live_.

## Stage derivation (proof, not opinion)

| Stage    | Done when                                                  |
| -------- | ---------------------------------------------------------- |
| Idea     | any successful tool run, or idea scored                    |
| Validate | idea validated + stress-tested (kill-my-idea / competitor) |
| Offer    | offer generated with pricing                               |
| Build    | customer plan / go-to-market system written                |
| Launch   | first leads saved                                          |
| Revenue  | first deal won                                             |

`readyForNova` = Build done **and** Launch done → the handoff hero appears and
the sidebar door upgrades from a quiet link to "Ready to operate".

## Product identity tokens

`[data-product="nova"]` on the app shell (set in `src/routes/app.tsx` from
workspace mode) remaps the semantic accent tokens — `--primary`,
`--primary-soft/-border/-hover/-active/-glow`, `--ring`, `--accent`, sidebar
accents — to `--nova-accent` (cyan, per-theme values in `styles.css`).
Components keep using `var(--primary)` and inherit the correct product
identity for free. Launchpad keeps the default violet.

## Mobile

Two tab bars (`MobileTabBar`), one per product, mirroring each sidebar:

- Launchpad: Home · Missions · Research · Memory · Ask
- Nova: Home · CRM · Pipeline · Tasks · Ask

## Copy system

Concise, operational, plain. Say _"fix your offer"_, _"4 leads need
attention"_, _"pipeline is live"_ — never _"explore features"_ or
_"configure modules"_. Launchpad speaks in proof and momentum; Nova speaks
in status and control.

## Validation test (run before shipping any screen)

Clarity in 10s · unmistakably Launchpad **or** Nova · obvious next action ·
creates momentum · closed loop visible · adaptive not trendy · beautiful on
a phone · an ambitious 18-year-old instantly gets it · doesn't look like
standard SaaS.
